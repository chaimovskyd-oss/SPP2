SPP2 — Batch Design Production Variable

אפיון מלא למודול ייצור סדרתי מעיצוב משתנה: יצירת תבניות מתוך Free Mode, סימון שכבות כמשתנות, ספריית תבניות, קישור ל־Product Library, אשף ייצור, בדיקות, ופרומפטים מחולקים ל־Phases.

תוכן עניינים

1. חזון והחלטות ליבה 2. Flow מלא למשתמש 3. יצירת תבנית מתוך Free Mode 4. מה נשמר ומה לא נשמר 5. ספריית תבניות ייצור 6. קישור דו־כיווני ל־Product Library 7. אשף הייצור 8. המסמך שנוצר 9. מודל נתונים מוצע 10. בדיקות ו־Acceptance Criteria 11. חלוקה ל־Phases 12. פרומפטים מלאים

1. חזון והחלטות ליבה

המודול מיועד להפוך עיצוב אחד לתהליך ייצור סדרתי: לדוגמה עיצוב לתיק, קלמר, בקבוק, מגנט, תמונת מחזור, מדבקת שם או מוצר אישי אחר. המשתמש מעצב פעם אחת, מסמן איפה התמונה והשם משתנים, ואז מייצר עותקים לפי רשימת תמונות/שמות.

החלטה 1

לא בונים עורך תבניות נפרד.
מעצבים רגיל ב־Free Mode, ואז מסמנים שכבות כ־Variable.

החלטה 2

Template הוא מסמך SPP רגיל + metadata.
כל מנוע השכבות, הטקסט, המסיכות וההדפסה נשארים קיימים.

החלטה 3

Generate יוצר מסמך רגיל.
אחרי הייצור כל דף ניתן לעריכה חופשית, בלי קשר חי לתבנית.

החלטה 4

התוכן מתחלף, העיצוב נשמר.
שם ותמונה משתנים; מיקום, סגנון, מסיכה, אפקטים ו־Blend Mode נשמרים.

כלל הזהב: Design first. Mark variables later. Generate detached editable pages.

2. Flow מלא למשתמש

  1. המשתמש פותח Free Mode או מוצר מתוך Product Library.
  2. מעצב כרגיל: רקע, מסיכה לתמונה, טקסט שם, אפקטים, פריסטים, קווים, צללים וכו’.
  3. בוחר שכבת תמונה/מסיכה ומסמן אותה כ־Variable Image Slot.
  4. בוחר שכבת טקסט ומסמן אותה כ־Variable Text Slot.
  5. שומר כ־Batch Production Template.
  6. ניגש דרך Hub למסך Batch Production.
  7. בוחר תבנית מתוך הספרייה.
  8. מעלה תמונות.
  9. המערכת מזהה שמות משמות קבצים ומציגה טבלה לעריכה.
  10. המשתמש מאשר.
  11. נוצר מסמך SPP רגיל עם עמוד לכל רשומה.
  12. המסמך נפתח ב־Free Mode לעריכה, הדפסה או Export.

3. יצירת תבנית מתוך Free Mode

ה־Flow הקריטי: המשתמש לא צריך להבין “תכנות תבניות”. הוא עובד רגיל, ורק בסוף אומר למערכת אילו שכבות הן משתנות.

פעולות UI מומלצות

מצב פעולה תוצאה
בחירת שכבת תמונה/מסיכה Batch Variable → Enable → Variable Image השכבה מקבלת Field ID כמו photo
בחירת שכבת טקסט Batch Variable → Enable → Variable Text השכבה מקבלת Field ID כמו name
שמירה Save as Batch Template נשמרת תבנית בספריית Batch Templates

אינדיקציה ויזואלית

  • Badge קטן על שכבה משתנה: VAR IMG או VAR TXT
  • בפאנל שכבות: אייקון קטן ליד השכבה.
  • ב־Properties Panel: אזור בשם Batch Variable Settings.

שמות שדות מומלצים ל־V1

  • photo — תמונה ראשית.
  • name — שם/כיתוב ראשי.

4. מה נשמר ומה לא נשמר

עיקרון בסיסי

יש להפריד בין תוכן משתנה לבין עיצוב קבוע. התוכן מתחלף בכל רשומה, אבל כל ההגדרות העיצוביות נשמרות.

טקסט משתנה

לא נשמר כערך קבועכן נשמר
הטקסט עצמו, למשל “נועה כהן”. בתבנית זה Placeholder בלבד. מיקום, גודל תיבה, פונט, גודל פונט, צבע, יישור, RTL, ריווח, Stroke, Shadow, Glow, Preset, Opacity, Rotation, Scale, Blend Mode, Layer Order, Auto Resize, Min Font Scale.

תמונה/מסיכה משתנה

לא נשמר כברירת מחדלכן נשמר
קובץ תמונת הדוגמה, קרופ ידני ספציפי, כוונוני תמונה ספציפיים כמו Brightness/Contrast/Exposure, Background Removal, ריטוש ידני. מיקום המסיכה, גודל, צורה, Rotation, Scale, Opacity, Layer Order, Blend Mode, Stroke/Shadow של מסגרת, Fit Mode, Cover/Contain, Smart Crop On/Off, Safe Area, Clip Behavior.

כוונוני תמונה יכולים לעבור רק אם המשתמש מסמן במפורש: “Apply template image adjustments to all generated images”.

5. ספריית תבניות ייצור

יש לגשת לתבניות ממיקום נפרד ב־Hub: Batch Production. זה לא עורך חדש, אלא אזור ניהול והפקה.

כרטיס תבנית צריך לכלול

  • Preview של התבנית.
  • שם התבנית.
  • גודל / יחס / Orientation.
  • מספר שדות משתנים.
  • Tags או מוצר מתאים.
  • תאריך עדכון אחרון.

כפתורים חובה

Produce
פותח אשף ייצור מהתבנית.
Edit
פותח את התבנית ב־Free Mode לעריכה.
Duplicate
יוצר עותק עצמאי לשינוי מידות/מוצר.
Delete
מחיקת תבנית אחרי אישור.

Duplicate — שימוש חשוב

לדוגמה: יש עיצוב מוצלח לתיק גן. המשתמש לוחץ Duplicate, משנה שם ל“קלמר תואם”, משנה פרופורציות ומיקומי טקסט/מסיכה, ושומר כתבנית חדשה.

6. קישור דו־כיווני ל־Product Library

ב־V1 אפשר לבנות קישור פשוט וחכם לפי מידות/יחס. אין צורך במערכת כבדה.

בתבנית Batch

{
  compatibleProductIds: ["kids_bag", "pencil_case"]
}

במוצר Product Library

{
  compatibleBatchTemplateIds: ["template_123", "template_456"]
}

התאמה אוטומטית

  • בדיקת Width/Height.
  • בדיקת Ratio.
  • בדיקת Orientation.
  • Tolerance של 3%–5%.
  • בעתיד: Bleed / Safe Area / Print Zones.

חשוב לקשור לפי productId, לא לפי שם מוצר, כי שמות יכולים להשתנות.

8. המסמך שנוצר

לאחר Generate נוצר מסמך חדש רגיל ונפתח ב־Free Mode. אין קשר חי לתבנית.

מה נשמר במסמך שנוצר

{
  generatedBatch: {
    templateId: "template_123",
    templateName: "תיק גן עם שם ותמונה",
    generatedAt: "2026-05-18T10:00:00Z",
    recordCount: 38
  }
}

יתרונות

  • כל עמוד ניתן לעריכה עצמאית.
  • אפשר לתקן רק ילד אחד.
  • אפשר להוסיף אלמנטים ידנית.
  • אפשר למחוק או לסדר עמודים.
  • Print/Export משתמשים במערכת הקיימת.

9. מודל נתונים מוצע

Batch Template Metadata

type BatchTemplateMetadata = {
  enabled: boolean;
  templateId: string;
  templateName: string;
  description?: string;
  tags?: string[];
  compatibleProductIds?: string[];
  canvas: {
    width: number;
    height: number;
    unit: "mm" | "px";
    orientation: "portrait" | "landscape" | "square";
    ratio: number;
  };
  variableFields: BatchVariableField[];
  createdAt: string;
  updatedAt: string;
};

Variable Field

type BatchVariableField =
  | {
      id: string;
      type: "image";
      layerId: string;
      label: string;
      fitMode: "cover" | "contain" | "fill";
      smartCrop: boolean;
      preserveMask: boolean;
      applyImageAdjustmentsByDefault: boolean;
    }
  | {
      id: string;
      type: "text";
      layerId: string;
      label: string;
      sourceField: "name" | string;
      preserveTextStyle: true;
      autoResize: boolean;
      minFontScale: number;
    };

Batch Record

type BatchRecord = {
  id: string;
  imageFileId?: string;
  fields: {
    name?: string;
    [customField: string]: string | undefined;
  };
  status: "ready" | "warning" | "error";
};

12. פרומפטים מלאים

פרומפט ראשי — בסיס לכל הפרויקט

We are adding a new major module to SPP2 called Batch Design Production / Variable Design Mode.

This module allows the user to design once and generate many personalized design pages from images and names.

Important architecture decisions:
1. Do not create a separate template editor.
2. Templates are created inside the existing Free Mode editor.
3. The user designs normally, then marks existing layers as variable fields.
4. A Batch Template is a normal SPP document with additional batch metadata.
5. Generated output is a new normal SPP document opened in Free Mode.
6. Generated pages must be fully editable with all existing SPP2 tools.
7. There is no live link between generated pages and the original template.
8. Variable content is replaced; design properties are preserved.
9. Product Library integration should be simple and bidirectional, based mainly on product/template dimensions and aspect ratio.

Core V1 field types:
- Variable Image Slot
- Variable Text Slot

Variable Text:
- The actual placeholder text is replaced during generation.
- Preserve text design: font, size, color, alignment, RTL, stroke, shadow, effects, preset, opacity, rotation, scale, blend mode, layer order, auto-fit settings.

Variable Image:
- Replace the source image.
- Preserve mask/frame position, size, shape, rotation, scale, opacity, layer order, blend mode, fit mode, smart crop setting, clipping behavior.
- Do not preserve image-specific adjustments by default unless the user explicitly enables that option.

Template Library:
- Add a Batch Production area in the Hub.
- Display templates as cards with preview, name, size, field count, tags/products.
- Each template must have Produce, Edit, Duplicate, Delete.
- Edit opens the template in Free Mode.
- Duplicate creates an independent template copy with a new ID.

Generate:
- The wizard uploads images, extracts default names from filenames, allows editing in a table, then generates one page per record.
- The generated document opens in Free Mode as a normal editable document.
- Add generatedBatch metadata for reference only.

Build this gradually in phases. Preserve existing SPP2 architecture, stores, history, document model, export, and print systems.

Phase 1 Prompt — Data Model + סימון שכבות

Implement Phase 1 of Batch Design Production in SPP2.

Goal:
Allow users inside Free Mode to mark existing layers as Batch Variable fields.

Requirements:
1. Add batch metadata support to the document model.
2. Add variable field definitions:
   - image field
   - text field
3. Do not create new special layer types. Use existing image/mask/text layers and attach metadata/role to them.
4. Add UI in the selected layer properties panel:
   - Enable Batch Variable
   - Field Type: Image / Text
   - Field ID
   - Label
5. For image fields expose:
   - fitMode: cover/contain/fill
   - smartCrop: boolean
   - preserveMask: true
   - applyImageAdjustmentsByDefault: false
6. For text fields expose:
   - sourceField: name by default
   - preserveTextStyle: true
   - autoResize: true
   - minFontScale: 0.7
7. Add visual indicators:
   - badge on canvas/layer overlay
   - icon/tag in layers panel
8. If a variable layer is deleted, remove or invalidate its variable field metadata safely.
9. Ensure normal documents still work exactly the same.

Acceptance tests:
- Mark text layer as variable text.
- Mark image/mask layer as variable image.
- Save and reload document, metadata persists.
- Delete variable layer, no broken references remain.
- Existing Free Mode tools still work.

Phase 2 Prompt — שמירת תבניות וספריית Templates

Implement Phase 2 of Batch Design Production in SPP2.

Goal:
Create a Batch Templates Library and allow saving the current Free Mode document as a Batch Production Template.

Requirements:
1. Add "Save as Batch Template" action.
2. Validate before saving:
   - at least one variable field exists
   - variable field layerIds exist
   - field IDs are unique
3. Store template as a normal SPP document plus batchTemplate metadata.
4. Generate and save a preview thumbnail for the template card.
5. Add a new Hub entry: Batch Production.
6. Add a Batch Templates Library screen with template cards.
7. Each card should show:
   - preview
   - template name
   - canvas size / ratio / orientation
   - number of variable fields
   - compatible products if available
   - updated date
8. Add actions:
   - Produce
   - Edit
   - Duplicate
   - Delete
9. Edit opens the template in Free Mode.
10. Duplicate creates a fully independent copy with a new templateId and new updatedAt.
11. Delete asks for confirmation.

Acceptance tests:
- Save current design as template.
- Template appears in library.
- Edit opens it correctly.
- Duplicate creates independent copy.
- Deleting original does not delete duplicate.
- Produce button exists but may be placeholder until Phase 3.

Phase 3 Prompt — אשף העלאת תמונות וטבלת שמות

Implement Phase 3 of Batch Design Production in SPP2.

Goal:
Create the Produce wizard for a selected Batch Template.

Wizard steps:
1. Template summary
2. Upload images
3. Build/edit records table
4. Preview sample
5. Confirm generate

Requirements:
1. User can upload multiple image files.
2. Supported formats: jpg, jpeg, png, webp.
3. For each image create a BatchRecord.
4. Default name field is derived from filename without extension.
5. Clean simple filename separators:
   - underscores and hyphens become spaces
   - trim extra spaces
6. Show records in an RTL-friendly editable table.
7. Columns:
   - thumbnail
   - name
   - file name
   - status
   - actions
8. Allow:
   - edit name
   - delete record
   - replace image
   - reorder records if easy
9. Validate:
   - empty names show warning
   - missing image shows error
10. Add a small preview using the first record if possible.
11. Do not generate pages yet unless Phase 4 is already ready.

Acceptance tests:
- Upload 10 images.
- Names are extracted from filenames.
- Hebrew filenames display correctly.
- User can edit names.
- User can delete records.
- Wizard can proceed only when records are valid enough.

Phase 4 Prompt — Generate Engine

Implement Phase 4 of Batch Design Production in SPP2.

Goal:
Generate a new normal SPP document from a Batch Template and BatchRecords.

Architecture:
- Generated output must be a normal SPP document.
- One record creates one page in V1.
- No live link to the template.
- All generated pages open in Free Mode and remain fully editable.

Requirements:
1. Duplicate the template page once per valid record.
2. For each generated page:
   - Replace variable image field content with the record image.
   - Preserve mask/frame position, size, shape, rotation, scale, opacity, blend mode, layer order, fit mode.
   - Replace variable text field content with record.fields.name.
   - Preserve all text styling and effects.
3. Text auto-resize:
   - Only scale down if text exceeds bounds.
   - Respect minFontScale.
   - Do not scale up above original size.
4. Image fit:
   - Use cover by default.
   - Use smartCrop if available; otherwise center crop.
5. Add generatedBatch metadata:
   - templateId
   - templateName
   - generatedAt
   - recordCount
6. Open the generated document in Free Mode after generation.
7. The whole generation should be one history/logical action if possible.
8. Do not flatten layers.

Acceptance tests:
- 5 images create 5 pages.
- Each page has the correct image.
- Each page has the correct name.
- Text style is preserved.
- Image mask is preserved.
- Blend modes and opacity are preserved.
- User can edit any generated page manually.
- Print/export sees all generated pages.

Phase 5 Prompt — Product Library Link

Implement Phase 5 of Batch Design Production in SPP2.

Goal:
Add simple bidirectional compatibility between Batch Templates and Product Library products.

Requirements:
1. Add compatibleProductIds to Batch Template metadata.
2. Add compatibleBatchTemplateIds to Product metadata if appropriate.
3. Create automatic compatibility suggestion by:
   - matching width/height
   - matching aspect ratio
   - matching orientation
   - tolerance 3% to 5%
4. In Batch Template cards, show compatible products.
5. In Product Library, show compatible Batch Templates.
6. From a product card, allow:
   - Create Batch Template from product
   - Produce using compatible template
7. From a template card, allow:
   - link/unlink products manually
8. Use stable productId/templateId links, not product/template names.
9. Do not make this system blocking. Compatibility is helpful metadata, not a hard restriction.

Acceptance tests:
- Product with matching ratio suggests template.
- Template can be manually linked to product.
- Product shows linked templates.
- Duplicate template does not automatically overwrite links unless intentionally copied.
- Create template from product opens Free Mode with product canvas dimensions/guides.

Phase 6 Prompt — Polish, Errors, Performance

Implement Phase 6 polish for Batch Design Production in SPP2.

Focus:
UX, validation, performance, and reliability.

Requirements:
1. Add clear error states:
   - no variable fields
   - missing image field
   - missing text field if template expects name
   - broken layer reference
   - empty record name
   - failed image load
2. Add warnings, not blockers, when appropriate.
3. Optimize thumbnails:
   - do not store huge base64 unnecessarily
   - avoid rerendering all previews on every edit
4. Improve previews:
   - preview first record
   - optional preview first 3 records
5. Add Hebrew/RTL polish:
   - labels
   - table direction
   - text input alignment
6. Add template badges:
   - Batch Template Mode when editing template
   - Variable field badges on canvas/layers
7. Add save prompts:
   - if editing template and leaving with unsaved changes
8. Add logs/debug info only where useful.
9. Make sure Batch Production does not break existing Free Mode, Product Library, Print, Export, or history behavior.

Acceptance tests:
- Works with 1 image.
- Works with 50 images.
- Hebrew names work.
- Long names shrink correctly.
- Missing image shows understandable error.
- Broken template metadata does not crash.
- Generated document prints/exports normally.
- App remains responsive during wizard and generation.

Check specifically:
1. Are templates normal SPP documents with metadata, not a separate incompatible format?
2. Are generated outputs normal editable Free Mode documents?
3. Is there no live link that can unexpectedly change generated pages?
4. Are text styles fully preserved when replacing content?
5. Are masks and image placement rules preserved when replacing images?
6. Does Duplicate create an independent template with a new ID?
7. Are Product Library links based on IDs and ratio/dimensions, not names?
8. Are broken layer references handled safely?
9. Does print/export work with generated multi-page documents?
10. Are Hebrew/RTL names and text fields handled correctly?

Report any architecture risks before changing code.
