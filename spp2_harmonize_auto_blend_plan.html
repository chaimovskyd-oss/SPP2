תוכנית עבודה ל-SPP2: Harmonize Layer + Auto Blend Stack

מסמך עבודה להוספת שני כלי עיבוד מתקדמים ל-SPP2: כלי התאמת שכבה לסביבה, וכלי מיזוג שכבות/תמונות מוערמות. הדגש הוא על בנייה מדורגת, לא הרסנית, ניתנת לבדיקה, ושלא תפגע בכלים קיימים.

1. תקציר והמלצה

ההמלצה היא לא לנסות לבנות מיד “פוטושופ מלא”, אלא להתחיל משני כלים שימושיים מאוד לדפוס:

Harmonize Layer

כפתור שמקבל שכבה נבחרת, מנתח את הסביבה שלה, ומתאים לה צבע, בהירות, קונטרסט, רוויה, טמפרטורת צבע, חדות ורעש.

ריאלי מאוד שימושי במוקאפים לא חייב AI

Auto Blend Stack

כלי עצמאי שמקבל כמה תמונות/שכבות וממזג אותן לפי מטרה: Focus Stack, Exposure Blend, או מיזוג פנורמה בסיסי.

דורש OpenCV עדיף כמודול נפרד בנייה מדורגת
פיצ׳ר גרסה ראשונה מומלצת קושי ערך ל-SPP
Harmonize Layer התאמות צבע/אור/קונטרסט/רוויה לפי אזור רקע בינוני גבוה מאוד
Focus Stack בחירת האזורים החדים מכל תמונה בינוני+ גבוה לצילומי מוצרים
Exposure Blend Mertens exposure fusion דרך OpenCV בינוני נחמד לצילומים קשים
Panorama Blend יישור + תפר + מיזוג בסיסי קשה פחות דחוף
ההמלצה הפרקטית: להתחיל מ-Harmonize Layer, כי הוא הכי מתאים לעבודה היומיומית שלך: הדבקת אלמנטים, מוקאפים, קולאז׳ים, תמונות למוצרים, ושילוב תמונות שהגיעו ממקורות שונים.

2. עקרונות בנייה

  • לא הורסים מקור: כל פעולה צריכה להיות לא הרסנית ככל האפשר, או ליצור שכבה חדשה/עותק.
  • מודול נפרד: לא להכניס לוגיקה כבדה לתוך EditorScreen ישירות.
  • Fallback: אם Python/OpenCV/AI לא זמין, התוכנה לא קורסת אלא מציגה הודעה ברורה.
  • Preview לפני Apply: תמיד עדיף תצוגה מקדימה עם מחוון strength.
  • Undo מלא: כל פעולה חייבת להיכנס להיסטוריית Undo/Redo.
  • שמירת פרויקט: פעולה שיוצרת שכבה/פילטר צריכה להישמר בפרויקט בלי לשבור טעינה ישנה.
  • עיבוד כבד ב-sidecar: OpenCV, מודלים או חישובים כבדים — Python sidecar.

3. כלי Harmonize Layer

מטרת הכלי

המשתמש בוחר שכבה, לוחץ “התאם לסביבה”, והמערכת משנה את מראה השכבה כך שתשתלב טוב יותר עם הרקע או עם האזור שבו היא נמצאת.

מה הכלי צריך לדעת לעשות בגרסה ראשונה

  • לזהות את שכבת היעד.
  • לרנדר את שכבת היעד כ-PNG שקוף.
  • לרנדר את הרקע/הקנבס בלי שכבת היעד.
  • לחשב אזור סביב השכבה — margin של למשל 20–80px.
  • לנתח צבע, בהירות, קונטרסט ורוויה של סביבת הרקע.
  • לנתח את אותם נתונים בשכבה עצמה.
  • להחיל תיקון הדרגתי לפי strength.

התאמות מומלצות

התאמה מה היא עושה עדיפות
Brightness Match מתאים את רמת הבהירות הממוצעת של השכבה לסביבה. גבוהה
Contrast Match מקרב את הפער בין כהים לבהירים לרקע. גבוהה
Saturation Match מתאים רוויה כדי שהתמונה לא תיראה מודבקת מדי. גבוהה
Temperature Match מקרר או מחמם את השכבה לפי הרקע. גבוהה
Noise / Grain Match מוסיף מעט רעש אם הרקע מצולם והשכבה נקייה מדי. בינונית
Edge Softening מרכך מעט את גבולות האובייקט כדי להפחית תחושת הדבקה. בינונית
Shadow Tint שינוי קל בגוון הכהים כדי להתאים לתאורת הסביבה. מתקדמת

מצבי עבודה מוצעים

Auto

הכלי מנתח לבד את הסביבה ומציע תיקון.

Manual Fine Tune

המשתמש יכול לכוון בהירות, קונטרסט, רוויה וחום/קור.

Apply as Adjustment Layer

במקום לשנות את הפיקסלים, נוצרת שכבת התאמה מעל השכבה.

Pipeline טכני מוצע

  1. React שולח ל-sidecar:
    • PNG של השכבה הנבחרת עם שקיפות.
    • PNG של הקנבס/רקע בלי השכבה.
    • Bounding box של השכבה בקואורדינטות קנבס.
    • Strength והגדרות משתמש.
  2. Python מנתח את האזור סביב השכבה.
  3. Python מחשב פרופיל צבע של רקע ושל שכבה.
  4. Python מחזיר:
    • preview PNG
    • diagnostics JSON
    • recommended values
  5. React מציג Preview.
  6. Apply יוצר שכבה חדשה או adjustment non-destructive.
חשוב: בגרסה ראשונה לא חייבים AI. אפשר להגיע לתוצאה שימושית מאוד עם OpenCV / Pillow / NumPy בלבד.

4. כלי Auto Blend Stack

מטרה

כלי שמקבל כמה שכבות/תמונות דומות ומפיק תמונה אחת טובה יותר. בניגוד ל-Harmonize, זה לא כלי לשכבה אחת אלא לכלי ייעודי שמטפל בסטאק.

מצבים מומלצים

מצב שימוש רמת קושי המלצה
Focus Stack צילום מוצר מכמה פוקוסים — לוקחים מכל תמונה את האזור החד. בינוני+ מומלץ
Exposure Blend כמה חשיפות שונות — מיזוג לתמונה מאוזנת יותר. בינוני מומלץ
Clean Stack כמה תמונות דומות — הפחתת רעש/אנשים/אובייקטים משתנים. בינוני+ מעניין בהמשך
Panorama Blend חיבור תמונות רחבות. קשה לא שלב ראשון

Focus Stack — אלגוריתם בסיסי

  1. קבלת כמה תמונות.
  2. יישור תמונות לפי הראשונה — align.
  3. חישוב sharpness map לכל תמונה בעזרת Laplacian.
  4. לכל פיקסל/אזור בוחרים את התמונה הכי חדה.
  5. מחליקים את המסיכה כדי שלא יהיו קצוות קשים.
  6. ממזגים לתוצאה אחת.
  7. מחזירים preview ותמונת תוצאה.

Exposure Blend — אלגוריתם בסיסי

  1. קבלת כמה תמונות בחשיפות שונות.
  2. יישור בסיסי.
  3. שימוש ב-OpenCV MergeMertens.
  4. שמירת תוצאה ב-16bit או PNG איכותי אם אפשר.
  5. החזרה ל-SPP כשכבה/עמוד חדש.

Clean Stack — רעיון עתידי

אם מצלמים כמה תמונות מאותה נקודה, אפשר להשתמש ב-median/average חכם כדי להפחית רעש או להעלים אובייקטים שעברו בפריים. זה פחות דחוף, אבל יכול להיות כלי מאוד מגניב.

9. פרומפטים לקלוד

פרומפט 1 — תשתית Harmonize בלבד

אני רוצה להוסיף ל-SPP2 כלי חדש בשם Harmonize Layer, אבל בשלב הזה לבנות רק תשתית בטוחה בלי לשנות פיקסלים בפועל.

מטרה:
כאשר נבחרת שכבת תמונה ב-Editor, יופיע כפתור "התאם לסביבה".
לחיצה עליו תפתח פאנל צד קטן עם:
- כפתור "צור Preview"
- מחוון strength
- מקום להצגת preview
- הודעות שגיאה/סטטוס

דרישות טכניות:
1. אל תשנה את מבנה הפרויקט הקיים מעבר למינימום.
2. אל תכניס לוגיקה כבדה לתוך EditorScreen.
3. צור מודול חדש תחת features/harmonize.
4. צור טיפוסים מסודרים עבור בקשת harmonize ותשובה.
5. צור פונקציה שמרנדרת את השכבה הנבחרת ל-PNG שקוף.
6. צור פונקציה שמרנדרת את הקנבס בלי השכבה הנבחרת.
7. שלח ל-sidecar או ל-mock function את:
   - layer PNG
   - background PNG
   - bbox
   - options
8. בשלב הזה החזר preview זהה למקור כדי לבדוק את הזרימה.
9. ודא שיש טיפול בשגיאות, loading state, ו-timeout.
10. ודא שאין שינוי קבוע בשכבה עד שהמשתמש לוחץ Apply.

חשוב:
- לא לשבור Undo/Redo.
- לא לשבור autosave.
- לא לשנות כלים קיימים.
- אם אין שכבה נבחרת או השכבה אינה תמונה, הצג הודעה פשוטה.

פרומפט 2 — מימוש Harmonize בסיסי ב-Python

עכשיו אני רוצה לממש את הצד של Python עבור Harmonize Layer ב-SPP2.

מטרה:
לקבל PNG של שכבה עם alpha, PNG של רקע/קנבס בלי השכבה, bbox של השכבה, ואפשרויות משתמש.
להחזיר PNG מתוקן שבו השכבה מותאמת לסביבת הרקע.

נא לבנות מודול python-sidecar/tools/harmonize_layer.py.

דרישות:
1. השתמש ב-Pillow, NumPy ואם צריך OpenCV.
2. קרא את layerPngBase64 ואת backgroundPngBase64.
3. השתמש ב-bbox כדי לדגום אזור רקע סביב השכבה.
4. התעלם מפיקסלים שקופים בשכבה.
5. חשב:
   - brightness mean/std
   - saturation mean
   - basic color temperature
   - contrast estimate
6. בצע תיקון עדין לפי strength:
   - brightness
   - contrast
   - saturation
   - temperature
7. שמור alpha מקורי.
8. החזר previewPngBase64.
9. החזר diagnostics JSON עם הערכים שחושבו.
10. אם יש שגיאה, החזר ok:false עם reason ברור.

חשוב:
- לא לבצע תיקונים קיצוניים.
- strength 0 מחזיר תמונה מקורית.
- strength 1 מחיל תיקון מלא אבל עדיין עדין.
- חייב לעבוד גם על תמונות גדולות בלי לקרוס; אם צריך, בצע downsample לניתוח בלבד.

פרומפט 3 — Apply לא הרסני

אני רוצה להוסיף Apply ל-Harmonize Layer בצורה בטוחה ולא הרסנית.

דרישות:
1. אחרי יצירת preview, כפתור Apply יאפשר:
   א. יצירת שכבה חדשה מעל המקור עם התוצאה.
   ב. החלפה של השכבה הנבחרת רק אם המשתמש בוחר במפורש.
2. ברירת מחדל: צור שכבה חדשה.
3. השכבה החדשה תקבל שם: "Harmonized - [שם השכבה המקורית]".
4. הפעולה חייבת להיכנס ל-Undo/Redo.
5. autosave צריך לשמור את התוצאה כמו שכבת תמונה רגילה.
6. אל תשנה את מנגנון טעינת הפרויקטים הקיים אלא אם חייבים.
7. אם התמונה גדולה, השתמש במנגנון assets הקיים ולא ב-base64 בתוך state אם זה לא נכון לפרויקט.

פרומפט 5 — Focus Stack אמיתי

אני רוצה לממש Focus Stack אמיתי עבור Blend Stack ב-SPP2 דרך Python sidecar.

נא לבנות python-sidecar/tools/blend_stack.py עם mode focus-stack.

דרישות:
1. קבל רשימת תמונות PNG/JPEG כ-base64.
2. המר ל-NumPy/OpenCV.
3. אם align=true, יישר תמונות לפי הראשונה בעזרת ORB/ECC או fallback פשוט.
4. חשב sharpness map לכל תמונה בעזרת Laplacian.
5. צור מסיכת בחירה לפי התמונה החדה ביותר בכל אזור.
6. החלק את המסיכות כדי למנוע תפרים קשים.
7. מזג לתוצאה אחת.
8. החזר resultPngBase64.
9. החזר debug JSON:
   - כמה תמונות עובדו
   - אילו תמונות נכשלו
   - האם alignment הצליח
10. אם התמונות שונות מדי, החזר אזהרה במקום לקרוס.

חשוב:
- לשמור על זיכרון.
- לעשות resize פנימי אם התמונות ענקיות, אבל להחזיר איכות סבירה.
- לא להפיל את React אם יש שגיאה.
