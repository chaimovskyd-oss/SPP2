SPP2 – Smart Print Prepare

אפיון מקיף לפיצ'ר הכנה חכמה לתמונות לפני דפוס: ניקוי צילומי מסך, תיקון צבע בסיסי, התאמה למידה, קרופ חכם לפי פנים, פריסטים עיצוביים, בדיקה ידנית, דוח סופי ושילוב מלא בכלי SPP.
המטרה: “קיבלתי 150 תמונות מהלקוח — תכין אותן לעבודה.”

1. חזון הפיצ'ר

לא עוד כלי תיקון קטן
זה מנוע הכנה מרכזי שמטפל בתמונות לפני שהן נכנסות לקנבס, לפיתוח תמונות, לקולאז', לגריד, לתמונות מחזור או לכל כלי Batch אחר.
שימוש יומיומי אמיתי
לקוחות שולחים צילומי מסך, תמונות חשוכות, תמונות עם צבע מוזר, תמונות שלא מתאימות ליחס ההדפסה, ותמונות באיכות לא אחידה.
שתי רמות שליטה
מצב אוטומטי מלא לעבודה מהירה, ומצב בדיקה ידנית למקרים שבהם רוצים לשלוט בקרופ, צבע, פריסטים ואישור לכל תמונה.

שם מוצע

הכנה חכמה לדפוס Smart Print Prepare הכן תמונות לעבודה

2. שני מצבי שימוש

א. כלי עזר עצמאי

נמצא באזור כלי עזר. המשתמש גורר תיקייה או הרבה תמונות, בוחר פעולות, מפעיל, מקבל קבצים מתוקנים ודוח.

מתאים למצב שבו רוצים להכין תיקייה שלמה לפני עבודה, או להעביר ללקוח תיקייה מתוקנת.

ב. חלק מהייבוא לכלים רגילים

בכל כלי שכולל גרירת תמונות לפני פתיחת הקנבס מופיעה אפשרות: בצע הכנה חכמה לפני פתיחה.

מתאים לפיתוח תמונות, קולאז', גריד, תמונות מחזור, Batch Products, כרטיסי ברכה ועוד.

3. Flow משתמש מלא – כלי עזר עצמאי

בחירת תמונות – גרירת קבצים/תיקייה או כפתור בחירה.
בחירת פעולות – צ'קבוקסים קצרים: צילומי מסך, תיקון צבע, התאמה למידה, פריסט.
בחירת מצב עבודה – אוטומטי מלא או בדיקה ידנית לפני שמירה.
סריקה מקדימה – התוכנה מזהה בעיות אך לא משנה עדיין כלום.
תצוגת תוצאות צפויות – כמה צילומי מסך נמצאו, כמה תמונות דורשות צבע, כמה לא מתאימות למידה.
ביצוע – עיבוד כל התמונות לפי ההגדרות.
פלט ודוח – שמירה לתיקייה חדשה/בחירת תיקייה/יצירת פרויקט SPP/המשך לכלי אחר.

Flow מקוצר – כחלק מכלים רגילים

המשתמש בוחר כלי, למשל פיתוח תמונות.
המשתמש גורר 100 תמונות.
מופיע מסך קטן: “נבחרו 100 תמונות. לבצע הכנה חכמה?”
אם סומן — נפתח מסך אפשרויות מקוצר.
אוטומטי: הכלי נפתח עם תמונות מוכנות. ידני: קודם מסך אישור ואז המשך לכלי.

5. ImagePrepareEngine – מנוע פנימי מרכזי

חשוב לא לבנות את הפיצ'ר כקומפוננטה חד-פעמית. צריך לבנות מנוע פנימי אחד שכל הכלים משתמשים בו.

ImagePrepareEngine
├─ analyzeBatch(files, options)
├─ analyzeSingleImage(image)
├─ detectScreenshotArtifacts(image)
├─ detectColorProblems(image)
├─ detectFaces(image)
├─ computeSmartCrop(image, targetRatio, faces)
├─ applyTechnicalFixes(image, analysis, options)
├─ applyDesignPreset(image, preset, strength, guardrails)
├─ exportPreparedImages(results, outputOptions)
└─ generateReport(results)

מבנה נתונים מוצע

type PrepareOptions = {
  removeScreenshotArtifacts: boolean;
  autoColorFix: boolean;
  sharpenSoftImages: boolean;
  qualityCheck: boolean;
  targetSize?: {
    enabled: boolean;
    width: number;
    height: number;
    unit: 'cm' | 'mm' | 'inch' | 'px';
    dpi?: number;
  };
  designPreset?: {
    enabled: boolean;
    presetId: string;
    strength: number; // 0-100
    protectFromOverProcessing: boolean;
  };
  mode: 'auto' | 'manual-review';
  output: 'new-folder-next-to-source' | 'choose-folder' | 'new-spp-project' | 'continue-to-current-tool';
  neverOverwriteOriginals: true;
};
type ImageAnalysis = {
  filePath: string;
  width: number;
  height: number;
  aspectRatio: number;
  isLikelyScreenshot: boolean;
  screenshotCrop?: CropRect;
  colorIssues: Array<'too_dark'|'too_bright'|'too_yellow'|'too_blue'|'too_red'|'low_contrast'>;
  faces: FaceBox[];
  recommendedCrop?: CropRect;
  qualityWarnings: Array<'low_resolution'|'soft'|'heavy_compression'|'faces_near_edge'>;
  confidence: number;
};

6. ניקוי צילומי מסך

בשלב ראשון אפשר לבצע מקומית בלי AI כבד, בעזרת OpenCV וחוקים גיאומטריים.

זיהוי פס עליון/תחתון
חיפוש אזורים בגובה אופייני עם צבע אחיד, ניגודיות נמוכה, או אייקונים קטנים.
יחסים נפוצים
זיהוי מידות אופייניות של מסכי טלפון: 9:16, 19.5:9, 20:9, צילומי מסך ארוכים.
חיתוך בטוח
במצב בטוח חותכים רק אזורים ברורים מאוד. אם הביטחון נמוך — מסמנים לבדיקה ידנית.

שני מצבי ניקוי

מצבמה עושהסיכון
בטוחמסיר רק פסי מערכת ברורים: שעה, סוללה, ניווט תחתון.נמוך מאוד
אגרסיבימנסה להסיר גם Header/Footer של וואטסאפ, אינסטגרם, פייסבוק וכדומה.בינוני, עדיף עם בדיקה ידנית
כלל חשוב: אם הזיהוי לא בטוח, לא חותכים אוטומטית. מסמנים “דורש בדיקה”. עדיף לא לתקן מאשר להרוס תמונה.

7. תיקוני צבע בסיסיים

זה צריך להיות תיקון טכני עדין, לא פילטר עיצובי. המטרה: להביא תמונה למצב הדפסה סביר.

Auto Levels
מתיחה מתונה של היסטוגרמה, עם הגנה מפני שריפה.
White Balance
Gray World / Simple WB / תיקון טמפרטורה לפי גוון כללי.
CLAHE עדין
שיפור קונטרסט מקומי בלי להרוס פנים.
Vibrance
חיזוק צבעים מתונים בלי להקפיץ עור יותר מדי.
Sharpen עדין
Unsharp Mask חלש רק בתמונות רכות.
Skin Guard
הגנה בסיסית על גווני עור כדי לא להפוך אנשים לאדומים/כתומים מדי.

כלל פעולה

if imageLooksGood:
  do not auto-fix color
else:
  apply minimal technical correction
  compare before/after metrics
  if correction is too strong or suspicious:
    mark for manual review

9. פריסטים עיצוביים

אפשר להוסיף כבר בשלב ראשון חיבור לספריית הפריסטים הקיימת, אבל חייבים להפריד בין תיקון טכני לבין לוק עיצובי.

תיקון טכני

איזון לבן, חשיפה, קונטרסט בסיסי, ניקוי צילום מסך, קרופ חכם. מטרתו לתקן בעיות.

פריסט עיצובי

HDR, שחור לבן, Product Punch, Vintage, Duotone, Warm Look. מטרתו לתת סגנון.

סדר פעולות מומלץ

1. Decode image
2. Detect and crop screenshot artifacts
3. Compute target-size crop
4. Apply technical color correction only if needed
5. Apply optional design preset
6. Apply guardrails against over-processing
7. Export / continue to tool

UI מוצע לפריסטים

☐ החל פריסט עיצובי על כל התמונות [HDR Pop ▼] עוצמת פריסט: 70% ☑ הגן מפני תיקון מוגזם סדר פעולה: (●) תיקון טכני קודם, פריסט אחרי ( ) פריסט בלבד, ללא תיקון אוטומטי ( ) תיקון בלבד, ללא פריסט

Guardrails למניעת הגזמה

  • להגביל Saturation אם התמונה כבר קיבלה Vibrance.
  • להגביל Sharpen אם כבר הופעל חידוד טכני.
  • למנוע Contrast מוגזם אחרי Auto Levels.
  • לשמור גווני עור בטווח טבעי ככל האפשר.
  • אם המדדים קיצוניים אחרי פריסט — לסמן לבדיקה ידנית.

12. תוכנית יישום בשלבים

Phase 1 – גרסה שימושית ומהירה

  • UI כלי עזר עצמאי.
  • בחירת תמונות/תיקייה.
  • ניקוי צילום מסך במצב בטוח.
  • תיקון צבע טכני בסיסי רק כשצריך.
  • התאמה למידה עם קרופ מרכזי + זיהוי פנים אם כבר קיים/זמין.
  • חיבור לפריסטים קיימים עם עוצמה ו-Guardrails בסיסיים.
  • שמירה לתיקייה חדשה + דוח HTML/JSON.

Phase 2 – בדיקה ידנית ושילוב בכלים

  • מסך לפני/אחרי.
  • שליטה בקרופ ידני.
  • סליידרים בסיסיים לצבע.
  • פילטר “הצג רק בעייתיות”.
  • שילוב כ-checkbox בפיתוח תמונות, קולאז', גריד ותמונות מחזור.
  • המשך לכלי עם התמונות המתוקנות.

Phase 3 – חוכמה מתקדמת

  • זיהוי צילומי מסך לפי אפליקציות: WhatsApp, Instagram, Facebook.
  • Face grouping / Best photo selector בעתיד.
  • Mirror Edge / Generative Expand כשיחס התמונה לא מתאים.
  • Presets חכמים לפי סוג תמונה: אנשים, מוצר, נוף, מסמך.
  • שמירת פרופילים: “מצב חנות חדיש”, “מצב עדין”, “מצב אגרסיבי”.

Acceptance Criteria

בדיקהתוצאה נדרשת
100 תמונות רגילותהמערכת לא משנה תמונות ללא צורך.
20 צילומי מסך עם פסי מערכתלפחות רובם מזוהים ונחתכים במצב בטוח.
תמונות כהות/צהובותתיקון נראה טבעי ולא מוגזם.
התאמה ל-10x15 עם פניםפנים לא נחתכות, קרופ סביר.
פריסט HDR + תיקון צבעאין Saturation/Sharpen מוגזם בזכות Guardrails.
שמירהמקור לא נדרס לעולם.

13. פרומפטים מוכנים ליישום

פרומפט 1 – בניית מנוע ImagePrepareEngine

Implement a central ImagePrepareEngine for SPP2.
The engine must be reusable by the standalone helper tool and by all image-import workflows such as Photo Prints, Collage, Grid, Class Photos, and Batch Products.

Core requirements:
1. Never overwrite original files.
2. Analyze a batch before applying changes.
3. Detect likely screenshots and compute safe crop rectangles for top/bottom UI artifacts.
4. Detect basic color issues: too dark, too bright, too yellow, too blue, too red, low contrast.
5. Apply minimal technical corrections only when needed.
6. Support optional target size crop with face-aware smart crop.
7. Support optional design preset after technical correction.
8. Include guardrails to prevent over-processing when technical correction and presets are both active.
9. Return detailed per-image result objects and a batch summary report.
10. Keep the API clean and independent from the UI.

Create TypeScript types for PrepareOptions, ImageAnalysis, PrepareResult, BatchPrepareReport.
If Python/OpenCV is already available in the project, expose the heavy image operations through the existing Python sidecar. Otherwise create clean placeholders and fallback implementations.

פרומפט 2 – UI כלי עזר עצמאי

Create a new standalone helper tool screen in SPP2 called “הכנה חכמה לדפוס / Smart Print Prepare”.

UX requirements:
1. User can drag many images or a folder.
2. Show selected image count.
3. Show compact options:
   - remove screenshot artifacts
   - basic color correction only when needed
   - gentle sharpen for soft images
   - quality check
   - target size dropdown with custom size option
   - optional design preset dropdown from the existing preset library
   - preset strength slider
   - protect from over-processing checkbox
4. Mode selection:
   - automatic process and save
   - manual review before saving
5. Output selection:
   - new folder next to source
   - choose folder
   - create new SPP project
   - continue to another tool if launched from a workflow
6. After processing show report with counts and warnings.
7. Do not overwrite originals.
8. Use RTL Hebrew UI consistent with the existing SPP2 design.

פרומפט 3 – שילוב בכלי פיתוח/קולאז'/גריד

Integrate Smart Print Prepare into image import workflows.

When a user drags images before opening a canvas/tool, show an optional checkbox:
“בצע הכנה חכמה לתמונות לפני פתיחה”.

If checked, show a compact Smart Prepare panel with the same core options but adapted to the current tool.
Examples:
- Photo Prints 10x15 should auto-select target size 10x15.
- Collage should default to no destructive export and should pass prepared image data into the collage workflow.
- Grid/Class Photos should allow target size and face-aware crop.

Automatic mode should open the target tool immediately after processing.
Manual mode should open the review screen first, then continue to the target tool.
Ensure the original selected files remain unchanged.

פרומפט 4 – מסך בדיקה ידנית

Build a manual review screen for Smart Print Prepare.

The screen should show:
1. Thumbnail list of images with warning badges.
2. Filters: all, changed, screenshots, color issues, crop warnings, low resolution, manual review required.
3. Before/after preview for the selected image.
4. Per-image toggles:
   - remove screenshot crop
   - color correction
   - target crop
   - design preset
5. Crop editor with pan, zoom, reset, and face-safe overlay.
6. Basic color sliders: exposure, contrast, temperature, saturation, sharpen.
7. Buttons:
   - approve image
   - keep original
   - apply same settings to similar images
   - approve all safe changes
8. Keep the screen fast with thumbnail caching and low-res previews. Final export must use full quality.

פרומפט 5 – דוח HTML/JSON

Add report generation for Smart Print Prepare.

After processing, generate:
1. A user-friendly HTML report in Hebrew.
2. A machine-readable JSON report.

The report should include:
- total images scanned
- number of screenshots cleaned
- number of images color-corrected
- number of images cropped to target size
- number of images with design preset applied
- low-resolution warnings
- manual review warnings
- output folder path
- per-image original filename, output filename, actions applied, warnings, and confidence score

The report must be saved next to the exported images when using standalone helper mode.

  1. האם Phase 1 כולל בדיקה ידנית מלאה, או רק דוח + שמירה אוטומטית? המלצה: להתחיל אוטומטי + דוח, ואז ידני ב-Phase 2.

  3. האם פריסטים קיימים מיושמים כשכבות/פילטרים/מתכונים? צריך Wrapper אחד שמפעיל אותם על Bitmap Batch.
  4. איפה בדיוק יושב כלי העזר בתפריט SPP?
  5. באילו כלים משלבים קודם את checkbox ההכנה החכמה? המלצה: פיתוח תמונות וקולאז' ראשונים.

סיכום קצר

Smart Print Prepare צריך להיות מנוע מרכזי, לא כלי צדדי בלבד. הוא מקבל כמות גדולה של תמונות, מזהה בעיות נפוצות של לקוחות, מנקה צילומי מסך, מתקן צבע בעדינות, מתאים למידה, מאפשר פריסט עיצובי אחיד, נותן בדיקה ידנית כשצריך, ושומר פלט מסודר בלי לגעת במקור.

זה פיצ'ר דגל מעשי: פחות התעסקות, פחות טעויות, יותר מהירות, וממש מתאים לעולם של פיתוח תמונות, קולאז'ים ומתנות מודפסות.
