שילוב יוצר הברכות כמצב חדש ב־SPP2

תוכנית עבודה ופרומפט מוכן לקלוד — גרסה חדשה, מהירה, ללא Python, עם Auto Fit מלא לטקסט בעברית

מטרת המסמך

המטרה היא לבנות בתוך SPP2 מצב חדש ועצמאי בשם מצב ברכות, המבוסס רעיונית ותוכנית על יוצר הברכות הישן, אבל לא כהמרה ישירה שלו. המודול החדש צריך להיבנות ב־React/TypeScript, להשתמש בקנבס ובשכבות הקיימות של SPP2, ולהיות מהיר, יעיל, איכותי ומתאים לעבודה יומיומית בחנות.

חשוב מאוד: לא מבקשים מקלוד “להמיר את תוכנת הפייתון ל־React”. מבקשים ממנו לבנות מודול חדש ונקי ב־SPP2, ולהשתמש בכלי הישן רק כ־Reference וכמקור נתונים.

החלטות מוצר סופיות

נושא החלטה
אופי הכלי מצב עצמאי ב־SPP2, שנכנסים אליו מדף הבית.
התנהגות יצירה המצב יוצר עמוד חדש, ולא נכנס אוטומטית לעמוד קיים.
שפה עברית כברירת מחדל. תמיכה טובה ב־RTL היא חובה.
תמונות בתוך הברכה לא בגרסה הראשונה. אין photo slot כדי לא לסבך את המימוש.
רינדור טקסט הטקסט חייב להישאר שכבת טקסט אמיתית של SPP2, לא תמונה שטוחה.
ייצוא ושמירה דרך מערכות SPP2 הקיימות בלבד. לא לבנות Export/Save חדש.

הדגש המרכזי: Text Auto Fit Engine

זה הלב של הכלי. יוצר ברכות טוב חייב לדעת להכניס טקסטים באורכים שונים לתוך אזור עיצוב מוגדר, בלי שהמשתמש יצטרך להתחיל לשחק ידנית עם גדלים.

המנוע צריך לקבל

{
  text: string,
  boxWidth: number,
  boxHeight: number,
  fontFamily: string,
  maxFontSize: number,
  minFontSize: number,
  lineHeight: number,
  fontWeight?: string,
  direction: "rtl" | "ltr",
  align: "center" | "right" | "left"
}

המנוע צריך להחזיר

{
  fontSize: number,
  lines: string[],
  lineHeight: number,
  fits: boolean,
  overflowReason?: string
}

התנהגות נדרשת

  1. מדידת הטקסט באמצעות Canvas measurement בדפדפן.
  2. שבירת שורות לפי רוחב התיבה.
  3. תמיכה בעברית ו־RTL כברירת מחדל.
  4. הקטנת גודל הפונט בהדרגה עד שהטקסט נכנס לגובה התיבה.
  5. שמירה על טקסט מקורי ללא חיתוך שקט.
  6. אם הטקסט לא נכנס גם בגודל מינימלי — החזרת fits:false.
  7. הצגת אזהרה ב־UI כשהברכה ארוכה מדי.
אסור לפתור בעיית Auto Fit על ידי הפיכת הטקסט לתמונה. הטקסט חייב להישאר שכבת TextLayer רגילה ועריכה.

תבניות עיצוב נדרשות בגרסה הראשונה

מומלץ להתחיל עם 6–8 תבניות טקסט בלבד, ללא תמונה:

  • יוקרתי ונקי
  • מרגש ועדין
  • ילדותי ושמח
  • ברכת תודה
  • ברכה למורה/גננת
  • יום הולדת
  • ברכה זוגית/משפחתית
  • תבנית מינימליסטית ללא רקע

כל תבנית צריכה להגדיר

{
  id: string,
  name: string,
  recommendedSizes: string[],
  background?: TemplateBackground,
  titleBox: LayoutBox,
  bodyBox: LayoutBox,
  signatureBox?: LayoutBox,
  quoteBox?: LayoutBox,
  defaultFonts: {
    title: string,
    body: string,
    signature: string
  },
  colors: {
    background: string,
    title: string,
    body: string,
    signature: string
  },
  autoFit: {
    title: AutoFitSettings,
    body: AutoFitSettings,
    signature: AutoFitSettings
  }
}

פרומפט מלא לקלוד

אפשר להעתיק את כל הטקסט הבא לקלוד. מומלץ לתת לו קודם לבצע את שלב הסקירה, ורק אחרי שהוא מחזיר סיכום נקודות חיבור — לתת לו להמשיך למימוש. 

אני רוצה לשלב ב-SPP2 מודול חדש בשם "מצב ברכות" / Blessing Mode.

יש לי כלי ישן בשם hadish_blessing_designer_v042 שנמצא בתיקיית reference בלבד.
הכלי הישן נכתב בפייתון, אבל הוא לא אמור להפוך לחלק פעיל מהתוכנה.

חשוב מאוד:
- אל תמיר את אפליקציית הפייתון אחד לאחד.
- אל תייבא קוד Python.
- אל תריץ Python.
- אל תבנה Canvas חדש.
- אל תבנה מערכת Export חדשה.
- אל תשכפל מנגנוני שמירה קיימים.
- אל תיצור אפליקציה פנימית נפרדת בתוך SPP2.
- אל תיגע במצבים קיימים מעבר לחיבור ניווט מינימלי ונקי.
- השתמש בכלי הישן רק כמקור אפיון, מקור נתונים, מבנה JSON, והבנת זרימת עבודה.

המטרה:
לבנות מודול React/TypeScript חדש ונקי בתוך SPP2, שמשתמש בקנבס, שכבות הטקסט, מערכת העמודים, מערכת השמירה והייצוא הקיימות של SPP2.

המודול צריך להיות מצב עצמאי:
- נכנסים אליו מדף הבית של SPP2.
- הוא יוצר עמוד ברכה חדש.

- הוא לא כולל תמונה בתוך הברכה בגרסה הראשונה.
- רוב השימוש הוא בעברית, לכן RTL ועברית הם ברירת המחדל.
