רקע בעיות עקרונות תיקון סט תיקונים בדיקות פרומפטים

1. רקע מערכת

SPP2 היא תוכנת דסקטופ מבוססת Electron / React / Konva, עם רכיבי עיבוד תמונה ו־Sidecar בפייתון. המערכת כוללת מצבים כמו קולאג', מסיכה, גריד, עריכת תמונות, הכנת הדפסה, PDF, מוצרים, שכבות, שמירה אוטומטית וייצוא.

מצב מסיכה ומצבי גריד/קולאג' הם מצבי container: הם לא רק תמונה רגילה, אלא מסגרת או קבוצת תאים שבתוכה יושבות תמונות. לכן חשוב להפריד בין התא עצמו לבין התמונה שבתוכו.

תא / Cell

מסגרת, חריץ או אזור שמגדיר גבולות. הוא יכול לקבל גבול, צל, ריווח, מסיכה וצורה.

frame mask slot container

תמונה בתוך תא

התוכן הגרפי. יש לה crop, zoom, offset, fill/fit, rotation, כוונוני צבע ו־Face Fit.

image transform adjustments asset

עיצוב תא

גבול, צל, צבע, עובי, רדיוס, ריווח. אלו לא צריכים להרוס את התמונה או לשנות crop.

border shadow spacing

חוזה בסיסי

תא הוא מסגרת. תמונה היא תוכן. כוונוני תמונה חלים על התמונה. גבול וצל חלים על התא. שינוי מרווחים, החלפת מסיכה או הוספת תמונות לא אמורים לנעול מחדש תאים או לאפס הגדרות משתמש.

2. פירוק הבעיות שהועלו

בעיה מה קורה היום מה צריך לקרות סיכון מרכזי
ברירת מחדל נעולה במצב מסיכה תאים/שכבות נעולים כברירת מחדל. תאים לא נעולים כברירת מחדל, ונשארים כך גם אחרי פעולות נוספות. Rebuild של מסיכה עלול לאפס שוב ל־locked.
ריווח ביחידה לא ברורה המשתמש לא יודע אם הערך הוא פיקסלים, מ״מ או משהו אחר. ברירת מחדל מ״מ, עם אפשרות ס״מ/אינץ׳. המרה שגויה עלולה לשנות גודל הדפסה.
אין עיצוב מסיכה גלובלי אין פקדים נוחים לגבול/צל שמשפיעים על כל התאים. תפריט צד עם גבול, צבע, עובי, צל וכו'. לא לצייר את העיצוב כחלק הרסני מהתמונה.
חיתוך צילום מסך מעוות תמונה השוליים השחורים יורדים אבל התמונה נמתחת/מתעוותת. Crop נקי בלבד, בלי מתיחה. שמירה על גודל layer מול שינוי גודל bitmap.
אין כוונוני תמונה במצב מסיכה/גריד אי אפשר לערוך בהירות/קונטרסט וכו' לתמונה בתוך תא. לשונית כוונוני תמונה לצד לשונית המצב. UI בלי pipeline אמיתי ייצור אשליית תיקון.
כפתור עריכת פריסה חסר השפעה במצב מסיכה הכפתור מופיע אבל לא עושה כלום. להסתיר או להשבית עם tooltip. עדיף להסתיר כרגע. לא לשבור מצבים אחרים שבהם הכפתור כן עובד.

3. עקרונות תיקון כדי לא ליצור בעיות חדשות

3.1 לא לאפס state של משתמש

פעולות כמו הוספת תמונות, שינוי ריווח, החלפת מסיכה, שינוי כמות תאים או רענון layout לא אמורות לאפס נעילה, crop, כוונוני צבע, מיקום פנימי של תמונה או הגדרות עיצוב.

3.2 לשמור יחידות פיזיות במודל אחיד

מומלץ לשמור spacing פנימי במילימטרים, ולהמיר רק לתצוגה. כך הייצוא וההדפסה נשארים צפויים. 

spacingMm = 5

displayValue:
  mm   => 5
  cm   => 0.5
  inch => 0.19685

onUserInput:
  mm   => value
  cm   => value * 10
  inch => value * 25.4

3.3 להפריד בין עיצוב תא לעיבוד תמונה

גבול וצל הם decoration של התא. בהירות וקונטרסט הם adjustment של התמונה. אין לערבב ביניהם.

3.4 Crop צילום מסך לא משנה יחס תמונה

הסרת שוליים שחורים היא פעולה של חיתוך גבולות. אסור לבצע scale לא אחיד. אם רוצים לשמור את ה־layer באותו גודל ויזואלי, צריך לבצע זאת עם scale אחיד או עדכון bounds מודע, אבל לא למתוח את התמונה.

3.5 UI חייב להיות מחובר לפעולה אמיתית

אם מוסיפים לשונית כוונוני תמונה, היא חייבת להשפיע בפועל על תמונות בתוך תאים, להישמר בפרויקט, לעבוד ב־undo/redo, autosave וייצוא.

4. סט תיקונים מסודר

תיקון 1 — מצב מסיכה: תאים לא נעולים כברירת מחדל

דרישה

  • ברירת מחדל של תאי מסיכה תהיה לא נעולה.
  • המצב הלא נעול יישמר גם אחרי הוספת תמונות, שינוי מרווחים, החלפת מסיכה ושינויים נוספים.
  • אין לאפס ל־locked אלא אם המשתמש נעל במפורש.

יישום מומלץ

const DEFAULT_MASK_CELL_LOCKED = false;

// בעת יצירת תא חדש:
cell.locked = userProvidedLockedValue ?? DEFAULT_MASK_CELL_LOCKED;

// בעת rebuild:
cell.locked = previousCell?.locked ?? DEFAULT_MASK_CELL_LOCKED;

נקודת זהירות

לבדוק שאין פונקציות normalize / rebuild / regenerate שמגדירות locked: true באופן קשיח.

תיקון 2 — ריווח במצב מסיכה עם יחידות ברורות

דרישה

  • ברירת מחדל: מילימטר.
  • אפשרויות: מ״מ, ס״מ, אינץ׳.
  • שמירה פנימית במ״מ.
  • שינוי יחידה לא משנה את המרווח בפועל.

יישום מומלץ

type Unit = "mm" | "cm" | "inch";

function toMm(value: number, unit: Unit): number {
  if (unit === "mm") return value;
  if (unit === "cm") return value * 10;
  return value * 25.4;
}

function fromMm(mm: number, unit: Unit): number {
  if (unit === "mm") return mm;
  if (unit === "cm") return mm / 10;
  return mm / 25.4;
}

תיקון 3 — כלי עיצוב למסיכות / תאים

דרישה

להוסיף בתפריט הצד של מצב מסיכה אזור עיצוב שמשפיע על כל התאים.

  • גבול פעיל/כבוי.
  • עובי גבול.
  • צבע גבול.
  • צל פעיל/כבוי.
  • צבע צל.
  • טשטוש צל.
  • עוצמת צל / opacity.
  • היסט צל X/Y.

State מוצע

maskStyle: {
  borderEnabled: boolean
  borderColor: string
  borderWidthMm: number

  shadowEnabled: boolean
  shadowColor: string
  shadowBlur: number
  shadowOpacity: number
  shadowOffsetX: number
  shadowOffsetY: number
}

נקודת זהירות

אם הגבול מצויר מעל התמונה, הוא עלול לכסות קצוות. אם הוא מצויר מחוץ לתא, צריך לוודא שהוא לא נחתך. עדיף לאפשר render עקבי: צל מאחורי התא, תמונה clipped, גבול מעל התא.

תיקון 4 — חיתוך צילום מסך בלי עיוות

דרישה

  • לזהות שוליים שחורים.
  • לבצע crop נקי.
  • לא לבצע stretch לתוכן הפנימי.
  • לא לבצע resize לא פרופורציונלי.
  • לשמור יחס תמונה של התוכן שנשאר.

אלגוריתם בסיסי

1. Load image bitmap/canvas.
2. Scan from top/bottom/left/right until non-black pixels are found.
3. Compute cropRect.
4. Create a new bitmap/canvas with cropRect width/height.
5. Draw original image region into new bitmap at 1:1 scale.
6. Replace asset or create new cropped asset.
7. Update layer bounds carefully:
   - either shrink layer to cropped aspect
   - or preserve visual width with proportional height
   - never stretch X and Y independently.

תיקון 5 — כוונוני תמונה במצב מסיכה / גריד

דרישה

להוסיף לשונית כוונוני תמונה לצד לשונית מצב מסיכה/גריד כאשר נבחרת תמונה בתוך תא.

  • בהירות.
  • קונטרסט.
  • רוויה.
  • שחור־לבן.
  • ספיה / חום אם קיים.
  • איפוס כוונונים.

חיבור Pipeline

selectedCell.image.adjustments = {
  brightness,
  contrast,
  saturation,
  grayscale,
  sepia
}

render:
  image asset
  + fill/crop transform
  + adjustments
  + cell clipping
  + cell border/shadow decoration

נקודת זהירות

אין להסתפק בהוספת UI. צריך לבדוק שהשינוי משפיע ב־Konva, נשמר ב־state, משתלב עם Undo/Redo ומופיע בייצוא.

תיקון 6 — הסתרת כפתור עריכת פריסה במצב מסיכה

דרישה

  • במצב מסיכה, אם הכפתור לא פעיל באמת — להסתיר אותו.
  • לא לפגוע במצב קולאג' או גריד אם שם הכפתור כן רלוונטי.
  • אם מחליטים להשבית במקום להסתיר, להציג tooltip ברור.

6. פרומפטים מוכנים לקודקס

פרומפט 1 — מיפוי לפני שינוי

Before implementing, inspect SPP2 mask/grid/collage related code.

Map:
1. Where mask mode state is created and normalized.
2. Where mask cells/layers get their locked state.
3. Where spacing is stored and displayed.
4. Which units are currently assumed for spacing.
5. Where mask cells are rendered in Konva.
6. How normal ImageLayer adjustments are implemented.
7. Whether cell images in mask/grid modes reuse the same adjustment pipeline.
8. Where screenshot black-border crop is implemented.
9. Where the Edit Layout button is rendered and which modes show it.

Do not change code yet.
Return the relevant files, functions, current flow, root causes, and a safe implementation plan.

פרומפט 2 — תאים לא נעולים כברירת מחדל

Fix mask mode default cell locking.

Requirements:
- In mask mode, cells/layers should be unlocked by default.
- Default locked state should be false.
- Preserve this behavior after adding images, changing spacing, replacing masks, rebuilding mask layout, changing mask settings, and switching modes.
- If the user explicitly locks a cell/layer, preserve that explicit state.
- Do not reset locked=true during normalize/recompute/rebuild.
- Keep backwards compatibility with existing project files.

Add or update tests/manual checks:
- New mask mode cells are unlocked.
- Adding images keeps cells unlocked.
- Changing spacing keeps cells unlocked.
- Replacing mask keeps cells unlocked unless user explicitly locked.

פרומפט 3 — יחידות ריווח במצב מסיכה

Improve mask mode spacing units.

Requirements:
- Display spacing in millimeters by default.
- Add a unit selector: mm, cm, inch.
- Store internal spacing in millimeters if possible.
- Changing display unit should convert the displayed value but keep the actual physical spacing unchanged.
- Editing the value in any unit should convert back to mm internally.
- Ensure export/print dimensions remain correct.
- Avoid breaking old project files that stored spacing without unit metadata.

Add a safe migration/default:
- If spacingUnit is missing, assume mm unless existing code clearly used another physical unit.

פרומפט 4 — עיצוב מסיכה גלובלי

Add mask-wide style controls in mask mode side panel.

Controls:
- Border enabled
- Border width
- Border color
- Shadow enabled
- Shadow color
- Shadow blur
- Shadow opacity/intensity
- Shadow offset X/Y

Behavior:
- These controls apply to all mask cells.
- Render shadow/border as cell decoration, not as destructive modification to the image.
- Do not affect image crop/fill/transform.
- Save style in project state.
- Support undo/redo and autosave.
- Render correctly in export.

Recommended render order:
1. Shadow behind cell shape.
2. Clipped image content.
3. Border above clipped image.

פרומפט 5 — תיקון חיתוך צילום מסך

Fix screenshot black-border crop distortion.

Current bug:
The crop removes black borders but distorts/stretches the image.

Required behavior:
- Detect black borders.
- Crop the bitmap/canvas to the detected content bounds.
- Do not non-uniformly resize the remaining content.
- Do not stretch the cropped content back into the old dimensions.
- Preserve the content aspect ratio.
- If layer bounds must be updated, update them proportionally.
- Internal pixels should only be cropped, not warped.

Manual test:
Use a screenshot with black bars.
Run crop.
Verify black borders are removed and the image content is not stretched horizontally or vertically.

פרומפט 6 — כוונוני תמונה במצב מסיכה / גריד

Add image adjustment access for images inside mask/grid/container modes.

Requirements:
- In mask mode, grid mode and similar cell-based modes, show an Image Adjustments tab next to the mode-specific tab when an image inside a cell is selected.
- Include controls for brightness, contrast, saturation, grayscale/black-white, sepia if available, and reset.
- Reuse the normal ImageLayer adjustment pipeline where possible.
- Adjustments must apply to the selected image inside the cell, not to the whole mask/grid.
- Adjustments must save, undo/redo, autosave, and export correctly.
- The render order should be:
  image asset -> fill/crop transform -> image adjustments -> cell clipping -> cell border/shadow decoration.
- Avoid re-rendering all cells when only one selected image adjustment changes if possible.

פרומפט 7 — הסתרת כפתור עריכת פריסה במצב מסיכה

Hide or disable the Edit Layout button in mask mode.

Requirement:
- The Edit Layout button currently appears in mask mode but has no useful effect.
- Hide it in mask mode to avoid user confusion.
- Do not remove or break Edit Layout in collage/grid modes where it works.
- If hiding is not feasible, disable it and show a tooltip: "עריכת פריסה אינה זמינה במצב מסיכה".
- Make sure TypeScript checks pass and no dead handlers cause errors.

פרומפט 8 — רגרסיה מלאה

Run a focused regression pass after the mask/grid fixes.

Check:
1. Create new mask project.
2. Verify cells are unlocked by default.
3. Add images and verify cells remain unlocked.
4. Change spacing in mm, cm, inch and verify physical spacing remains correct.
5. Add border and shadow, verify all cells update.
6. Select an image inside a mask cell and adjust brightness/contrast/saturation.
7. Save and reopen project.
8. Export and compare visual output.
9. Replace mask and confirm unlocked/default/user states are preserved.
10. Test screenshot black-border crop and confirm no distortion.
11. Verify Edit Layout button is hidden in mask mode but still available where supported.
12. Run TypeScript checks and relevant tests.

Return:
- files changed
- bugs fixed
- remaining risks
- any unsupported edge cases
