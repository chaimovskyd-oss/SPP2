SPP2 — Batch Collages / יצירת קולאז׳ים כמותית

אפיון מלא וסט פרומפטים לבניית מצב Batch Collages כחלק מאשף הקולאז׳ הקיים: יצירת עשרות קולאז׳ים בעמודים נפרדים באותו פרויקט, עם בחירת פריסה אוטומטית לפי ניקוד, שימוש במנוע הקולאז׳ הקיים, ואפשרות עריכה רגילה לאחר מכן.
יעד MVP: עד 50 קולאז׳יםכ־6 תמונות לקולאז׳ בממוצעכל קולאז׳ = עמוד נפרדשילוב בתוך אשף הקולאז׳ללא מסך Review ייעודי בשלב ראשון

ניווט

רקע ומבנה קייםהחלטות מוצרזרימת משתמשמודל נתוניםאשף Batchמנוע יצירהניקוד ובחירת פריסהSmart Cropעמודים ועריכהפרומפטים לבנייהבדיקותהמשך עתידי

1. רקע על מבנה הקולאז׳ הקיים ב־SPP2

מצב הקולאז׳ הנוכחי בנוי סביב רעיון נכון: קולאז׳ הוא ישות מנוהלת בתוך עמוד, ולא קנבס נפרד. כל קולאז׳ נשמר כ־CollageRule, עם רשימת תמונות, תאים מחושבים, שיוכים בין תמונות לתאים, והמרה לשכבות/פריימים רגילים על הקנבס.

CollageRule

ישות הקולאז׳ המנוהלת. שומרת עמוד יעד, משפחת פריסה פעילה, spacing/margin, cached slots, image pool, assignments ועוד.

CollageSlot

תא בתוך הקולאז׳. יכול להיות מלבני, פוליגוני, לב/מסיכה ועוד.

Image Assignments

המיפוי בין תמונה לבין תא, כולל assetId, slotId, fit/crop/transform, התאמות צבע ונתוני Smart Crop.

FrameLayer

בפועל, התאים נראים בקנבס כשכבות/פריימים רגילים עם metadata של קולאז׳.

עיקרון חשוב: Batch Collages לא צריך לבנות מנוע קולאז׳ חדש. הוא צריך להשתמש במנוע הקיים שוב ושוב — פעם לכל קבוצה/ילד — וליצור עמוד נפרד לכל תוצאה.

3. זרימת משתמש מוצעת

  1. המשתמש פותח את אשף הקולאז׳ הרגיל.
  2. בעמוד הראשון מופיע checkbox: יצירת קולאז׳ים מרובים.
  3. אם לא סומן — האשף ממשיך כרגיל.
  4. אם סומן — האשף עובר למצב Batch.
  5. המשתמש מגדיר גודל דף, כיוון, spacing, margin, רקע וסגנון תאים כמו באשף הרגיל.
  6. המשתמש מוסיף קבוצות: שם קבוצה/ילד, תמונות לקבוצה, וכפתור +.
  7. המשתמש בוחר אם להפעיל Smart Crop אוטומטי.
  8. המשתמש בוחר אם להשתמש רק בפריסות רגילות או בכל סוגי הפריסות.
  9. לוחץ צור קולאז׳ים.
  10. המערכת יוצרת עמוד לכל קבוצה, בוחרת פריסה לפי ניקוד, ומפעילה Smart Crop אם נדרש.
  11. בסיום נפתח פרויקט רגיל עם כל העמודים.
אין סביבת עריכה חדשה. לאחר היצירה, כל קולאז׳ מתנהג כמו קולאז׳ רגיל ב־SPP2.

4. מודל נתונים מוצע

Batch לא חייב להישמר כישות כבדה בתוך הפרויקט בשלב ראשון. אפשר להשתמש במבנה זמני באשף, ואז ליצור עמודים וקולאז׳ים רגילים.

type BatchCollageGroup = {
  id: string;
  name: string;
  assetIds: string[];
  status: "ready" | "processing" | "created" | "warning" | "error";
  pageId?: string;
  collageRuleId?: string;
  chosenLayoutFamily?: CollageLayoutFamily;
  chosenScore?: number;
  warnings?: string[];
  error?: string;
};

type BatchCollageSettings = {
  pageSetup: PageSetup;
  spacingMM: number;
  marginMM: number;
  allowedLayoutMode: "safeOnly" | "allLayouts";
  smartCropEnabled: boolean;
  preferMinimalCrop: boolean;
  maxCollages: number;
};

type BatchCollageJob = {
  id: string;
  groups: BatchCollageGroup[];
  settings: BatchCollageSettings;
  createdAt: string;
};
אפשר לשמור את ה־BatchJob רק ב־state של האשף. התוצאה הסופית היא עמודים וקולאז׳ים רגילים.

6. מנוע יצירה אוטומטי

Batch Collage Engine הוא שכבת orchestration. הוא לא מייצר פריסות בעצמו, אלא מפעיל את מנוע הקולאז׳ הקיים לכל קבוצה.

  1. צור עמוד חדש לפי ההגדרות הכלליות.
  2. ייבא/שייך את כל התמונות של הקבוצה כ־assets.
  3. צור image inputs לפי מידות התמונות.
  4. הפק הצעות פריסה דרך המנוע הקיים.
  5. סנן פריסות לפי safeOnly או allLayouts.
  6. נקד לפי התאמת תמונות ומינימום חיתוך.
  7. בחר את ההצעה הטובה ביותר.
  8. צור CollageRule לעמוד.
  9. סנכרן ל־FrameLayers.
  10. אם Smart Crop מסומן — הרץ לכל assignment.
  11. עדכן שם עמוד לפי שם הקבוצה.
לא ליצור מסך/קנבס חדש ל־Batch. התוצאה חייבת להיות עמודים רגילים עם קולאז׳ים רגילים.

Prompt 2 — הגדרת Batch Mode בתוך אשף הקולאז׳

Implement Batch Collage entry inside the existing Collage Wizard.

In the first step of the existing Collage Wizard, add a checkbox:
"Create multiple collages / יצירת קולאז׳ים מרובים"

If unchecked:
- existing single collage flow remains unchanged.

If checked:
- wizard switches into Batch Collage mode.

Batch mode must still reuse the existing collage wizard page setup controls:
- page size
- orientation
- spacing
- margin
- background
- edge/cell style if already supported
- smart crop checkbox
- allowed layout mode checkbox:
  "Use all collage layout types"
  default false

Default behavior:

- output opens in normal editor, not a special review screen.

Do not implement export features in this step.
Do not create a new editor.

Prompt 3 — UI לקבוצות תמונות

Implement the Batch Collage Groups step.

UI requirements:
Each group represents one final collage/page.

Each group card includes:
- group name input
- drag-and-drop area
- choose images button
- thumbnails of selected images
- remove image button
- remove group button

Below the list:
- "+ Add child/group" button
- "Create collages" button

Supported input:
- user can add images to each group manually
- drag and drop into a specific group
- if folder drag/drop is available in Electron/browser, support it gracefully
- if folder drop is not available, do not break; user can choose files manually

Important:
There is no fixed number of images per group.
Each group uses all images assigned to it.
Each group later becomes one independent collage with its own best layout.

Validation:
- warn if a group has zero images
- allow creation only if at least one group has images
- recommended max: 50 groups
- recommended average: 6 images per group
- show warning if user exceeds recommended range, but do not crash.

Prompt 4 — Batch Engine

Create a Batch Collage Engine as an orchestration layer.

Do not create a new collage layout engine.
Reuse the existing single-collage engine for every group.

For each BatchCollageGroup:
1. create a new page using the selected page setup
2. import/add all group images as document assets
3. create CollageImageInput[] from the assets
4. generate collage layout suggestions using existing engine
5. filter suggestions:
   - safeOnly by default
   - allLayouts only if checkbox is enabled
6. score suggestions with preference for minimal crop and image fit
7. choose the best suggestion
8. create a normal CollageRule for that page
9. sync it to normal FrameLayers
10. optionally run smart crop if enabled
11. name the page after the group name
12. store warnings if any

Output:
A normal SPP2 document with one page per group.

Do not create a review screen for MVP.
Do not implement special export for MVP.
The user will use existing print/export tools.

Prompt 5 — Safe Layout Filtering

Implement safe layout filtering for Batch Collages.

Default Batch behavior:
Use only stable/safe layout families.

Safe families should include only layouts that:
- adapt reliably to image count
- work with page size changes
- produce readable cells
- minimize crop risk
- do not require manual fixing in most cases

Suggested safe families:
- grid
- hero
- magazine / clean layouts
- simple frame if stable
- simple split/tree only if stable

Do not include creative/geometric/heart/circle/blob layouts by default.

If the user enables:
"Use all collage layout types"
then include all currently available layout families, but still reject layouts that fail validation.

Important:
Batch default should prioritize reliable output over creative output.

Prompt 6 — בחירת פריסה לפי מינימום חיתוך

Improve Batch layout selection scoring.

Batch Collages should prioritize:
1. minimal cropping
2. image-to-cell aspect compatibility
3. face/crop safety if available
4. readable cell sizes
5. layout score

Suggested score:
finalScore =
  layoutScore * 0.35 +
  imageFitScore * 0.30 +
  cropSafetyScore * 0.25 +
  readabilityScore * 0.10

If the project already has a scoring function, reuse and extend it instead of duplicating.

Rules:
- Do not choose a layout with tiny cells for a small group.
- Do not choose a layout that cuts faces badly if smart crop/crop risk data is available.
- If all creative layouts score poorly, fallback to grid/hero.
- Store chosen score per group for status/debug.
