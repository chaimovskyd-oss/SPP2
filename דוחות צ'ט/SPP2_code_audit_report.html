דוח סקירת קוד SPP2

סקירה סטטית של ה־ZIP שנשלח. הדוח מתמקד בנקודות שיכולות להסביר קריסות, איטיות, שמירה לא אמינה, חיבורים חלקיים בין מודולים, ובעיות שיכולות לבלבל מאוד כלי AI בזמן תיקונים.

חשוב: זו לא הרצה מלאה של התוכנה ולא פרופיילינג בזמן אמת. המסקנות מבוססות על קריאת הקוד, שמות קבצים, דפוסי שימוש, ונקודות חיבור שנמצאו בסריקה.

נבדקו כ־456 קבצים ב־ZIP React + Electron + Konva + Python engines מוקד מרכזי: יציבות, שמירה, זיכרון, ארכיטקטורה

תקציר מנהלים

הבעיה הגדולה ביותר בפרויקט היא לא באג יחיד, אלא דפוס חוזר: מידע כבד כמו תמונות, previews ו־PDF נשמרים כ־base64 בתוך state/document/localStorage. זה פוגע ב־autosave, ב־undo, בתבניות, ב־PDF ובביצועים כלליים.

בנוסף, יש בעיית “מקורות אמת כפולים”: קבצי Editor ישנים בתוך src, וגם electron/main.ts לצד electron/main.cjs שלא זהים. זה מסביר למה קלוד/קודקס עלולים “לתקן” מקום שלא באמת רץ.

הכיוון הנכון הוא לא להוסיף עוד פלסטרים, אלא לבצע סדר בשלושה צירים: AssetStore חיצוני, Autosave קל, ו־ניקוי/פיצול קבצי מקור גדולים וכפולים.

Redesign recovery restore UX. restoreRecoveryRecord should return a typed result with full/partial/missing-assets statuses. Open partial projects with placeholders and a relink flow instead of discarding invalid records silently.
