1. תקציר החלטות

מה חינם?

עבודה מקומית בסיסית: פתיחת תמונות, עימוד בסיסי, הדפסה בסיסית, קולאז׳ים פשוטים, שמירה מקומית, מעט תבניות חינמיות.

מה בתשלום חד־פעמי?

כלים מקומיים מתקדמים שלא עולים כסף בכל שימוש: תבניות קולאז׳ איכותיות, Batch, Variable Data, תבניות ייצור, Product Library, PDF Studio מקומי, Smart Arrange.

מה במנוי?

כל מה שיוצר לך עלות שוטפת: AI בענן, גיבוי וסנכרון בענן, שיתוף קישורים, אחסון, תבניות מתעדכנות אם יהיו בעתיד.

חוקים חשובים למערכת הקרדיטים

  • לא לשים מפתחות API בתוך Electron. כל AI עובר דרך השרת שלך.
  • לא לחייב על כשל טכני, אבל להגביל retry אוטומטי.
  • להציג לפני הפעולה: כמה קרדיטים יירדו וכמה נשאר.
  • לשמור לוג פעולות ו־credit_transactions.
  • לעשות cache לפי image_hash + tool + settings_hash.
  • לשלוח crop/preview/רזולוציה נמוכה כשאפשר כדי להוריד עלות.

ספקים מומלצים

התחלה נוחה

Supabase לניהול משתמשים, רישיונות, מסד נתונים, webhooks בסיסיים והרשאות.

אחסון משתלם

Cloudflare R2 לקבצי פרויקטים, תמונות, assets ותוצרים, במיוחד אם יהיו הורדות ושיתוף.

שילוב מומלץ: Supabase Auth/Postgres לניהול משתמשים ורישיונות + Cloudflare R2 לאחסון קבצים.

חוקים לשירות ענן

  • הענן מיועד לפרויקטים ונכסים של SPP2, לא ככונן גיבוי כללי.
  • קבצי AI זמניים נמחקים אחרי 7–30 יום.
  • היסטוריית גרסאות מוגבלת, למשל עד 10 גרסאות לפרויקט.
  • קובץ בודד מוגבל, למשל 500MB/1GB לפי תוכנית.
  • אם יש התנגשות בין מחשבים: פתח גרסת ענן / שמור כעותק / החלף גרסת ענן.

8. ארכיטקטורה טכנית מומלצת

הכלל החשוב ביותר

AI לא יוצא ישירות מהתוכנה לספקים חיצוניים. אין API keys בתוך Electron. כל פעולה עוברת דרך השרת שלך.

SPP2 Electron App
  ↓ login/token
SPP2 API Server
  ↓ checks entitlements / credits / device limit / cloud quota
AI Gateway / Cloud Service
  ↓
Fal / Replicate / OpenAI / Stability / Local workers
  ↓
Result stored in R2/Supabase and returned to app
